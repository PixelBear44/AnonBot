## Что это

Маленький Telegram-бот, который хранит ваши секреты за кодовыми фразами.
Придумываете фразу → прячете за ней текст → потом пишете эту фразу, и бот
показывает секрет под спойлером. Один человек может завести сколько угодно
пар «фраза → секрет».

По желанию всё шифруется вашим личным кодом, который бот **не хранит** —
тогда содержимое базы не прочитает даже тот, кто запустил бота (см.
[раздел про безопасность](#-безопасность-и-приватность) — там честно про
границы).

## Возможности

- 🗝 **Секреты по фразе** — пишете фразу, получаете секрет под `спойлером`.
- 🔥 **Сгорающие секреты** — `/once` создаёт секрет, который исчезает после первого показа.
- 🔑 **Шифрование личным кодом** — `/code` шифрует ваши секреты; бот код не хранит.
- 🛡 **Защита от перебора** — 5 неверных попыток `/unlock` → пауза.
- ⏱ **Автозакрытие** — код сам защёлкивается после бездействия.
- 🆘 **Паник-код** — введёте код-ловушку в `/unlock` — всё молча сотрётся.
- 🤫 **Скрытность** — на неизвестную фразу бот не отвечает вовсе.
- 🧹 **Ручная уборка** — `/clear` стирает переписку, `/wipe` удаляет всё.

## Команды

| Команда | Что делает |
|---|---|
| `/add` | спрятать секрет (спросит фразу, затем секрет) |
| `/once` | секрет, сгорающий после первого показа |
| *(написать фразу)* | достать секрет под спойлером |
| `/list` | список ваших фраз, удалить лишние |
| `/code КОД` | зашифровать секреты личным кодом (`/code off` — снять) |
| `/unlock КОД` | открыть доступ · `/lock` — закрыть |
| `/panic КОД` | код-ловушка: при вводе в `/unlock` стирает всё |
| `/clear` | стереть всю переписку (секреты остаются) |
| `/wipe` | удалить вообще всё |
| `/help` | помощь |

## 🔐 Безопасность и приватность

Честно о том, от кого это защищает, а от кого — нет.

**Защищено:**
- Данные в базе (Upstash) зашифрованы мастер-паролем (Fernet, ключ через PBKDF2·200k). Провайдер БД и сеть видят только шифротекст поверх TLS.
- С `/code` секреты дополнительно шифруются вашим кодом, которого нет ни в базе, ни в коде бота. Из базы их не достать без кода.

**НЕ защищено (важно понимать):**
- **Telegram Bot API — не end-to-end.** Все сообщения (фразы, секреты, коды) проходят через серверы Telegram открытым текстом.
- **Владелец сервера** при желании доберётся до секретов: без `/code` — напрямую мастер-паролем; с `/code` — перехватив ввод `/unlock` или прочитав память процесса. Серверный бот в принципе не может быть недостижим для того, кто его запускает.
- **Раскрытые секреты остаются в чате**, пока не сделаете `/clear`.
- Короткий `/code` (например `1234`) можно перебрать офлайн — используйте длинную случайную фразу.

**Вывод:** отличный личный тайник и защита «от посторонних глаз и от кражи базы», но не средство против оператора сервера или самого Telegram. Для максимума — сильный `/code` и регулярный `/clear`.

## 🚀 Запустить свой (бесплатно, 24/7)

Полная пошаговая инструкция — в [DEPLOY.md](DEPLOY.md). Кратко, всё на бесплатных тарифах без карты:

1. **[@BotFather](https://t.me/BotFather)** → `/newbot` → получить `BOT_TOKEN`.
2. **[Upstash](https://upstash.com)** → создать Redis → взять `UPSTASH_REDIS_REST_URL` и `UPSTASH_REDIS_REST_TOKEN`.
3. **[Render](https://render.com)** → New → Blueprint → выбрать форк этого репо → задать 4 переменные (`BOT_TOKEN`, `SECRET_BOT_PASSWORD` — придумать самому, `UPSTASH_*`) → Deploy.
4. **[cron-job.org](https://cron-job.org)** → пинговать адрес сервиса каждые 10 минут, чтобы Render не засыпал.

Готово — ссылка вида `https://t.me/ВАШ_БОТ` работает без вашего ПК.

> ⚠️ `SECRET_BOT_PASSWORD` — единственный ключ ко всем данным без личного `/code`. Сделайте его длинным и случайным и **не меняйте**: смена = старые данные не расшифруются.

## 🧑‍💻 Локальный запуск

```bash
pip install -r requirements.txt

# Linux/macOS
BOT_TOKEN=... SECRET_BOT_PASSWORD=... python secret_bot.py

# Windows PowerShell
$env:BOT_TOKEN="..."; $env:SECRET_BOT_PASSWORD="..."; python secret_bot.py
```

Без переменных бот спросит их при старте. Если не задать `UPSTASH_*`, секреты
лягут в локальный зашифрованный файл `secrets.enc` (удобно для теста).

Тесты:

```bash
python test_secret_bot.py
```
